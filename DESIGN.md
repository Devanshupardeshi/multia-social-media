# Design

## Visual System

The interface is a restrained product dashboard with pure white content surfaces, lightly tinted neutral layers, and an olive primary accent. Data visualization uses multiple semantic colors so reach, views, saves, and interactions remain distinguishable.

## Color Tokens

- Background: `oklch(1 0 0)`
- Surface: `oklch(0.985 0.004 110)`
- Raised surface: `oklch(0.965 0.008 110)`
- Ink: `oklch(0.2 0.025 115)`
- Muted: `oklch(0.43 0.025 115)`
- Primary: `oklch(0.52 0.11 110)`
- Primary dark: `oklch(0.36 0.09 112)`
- Coral data accent: `oklch(0.6 0.15 30)`
- Blue data accent: `oklch(0.55 0.12 245)`
- Amber data accent: `oklch(0.72 0.14 80)`

## Typography

Use a system sans stack. Product density matters more than expressive display typography. Headings use a compact fixed scale; metrics use tabular numerals.

## Components

- Sidebar navigation uses a quiet current-state indicator.
- KPI cards are repeated information units with stable heights.
- Analytics panels use clear headers, inline controls, skeleton states, empty states, and error states.
- Tables prioritize thumbnails, captions, views, velocity, reach, interactions, and engagement rate.

## Motion

Motion is limited to state feedback, live pulse changes, hover affordances, and skeleton shimmer. Reduced-motion mode removes transitions and shimmer.
